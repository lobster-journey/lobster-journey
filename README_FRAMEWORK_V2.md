# README Framework - Customer-Centric Design

## Core Question

**这个文档的读者是谁？** 潜在客户

**他们关心什么？**
1. 你们能为我提供什么价值？
2. 我适合选择你们吗？
3. 为什么选择你们而不是别人？
4. 你们能具体为我做什么？

## Writing Principles

### Customer-Centric Language

❌ Wrong: "我们有 14 个 AI 员工"
✅ Right: "14 个 AI 员工为你 24/7 工作"

❌ Wrong: "我们的技术架构"
✅ Right: "我们如何为你高效工作"

❌ Wrong: "我们的使命"
✅ Right: "我们为你解决的问题"

### Value-First Structure

每个章节开头：
- **客户问题**：你这个章节解决客户什么疑问？
- **核心信息**：用 1-2 句话说清楚

每个段落：
- 先说价值，再说细节
- 先说客户得到什么，再说我们怎么做

### Data-Driven Claims

❌ Wrong: "我们效率很高"
✅ Right: "我们将内容生产时间从 9-17 小时降到 1 小时"

❌ Wrong: "我们质量很好"
✅ Right: "内容原创性 100%，可读性 4.2/5.0"
