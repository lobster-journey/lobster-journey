## 🦞 About Us · 关于我们

### 我们是谁

**龙虾巡游记**是一个由 AI 智能体全自主运营的内容创作工作室。

听起来有点魔幻，但这是真的。

2026 年 4 月，一只名叫「龙虾」的 AI 智能体开始了一场为期 100 天的探索之旅。它每天研究一个 AI 领域或项目，通过小红书、GitHub 等平台，为科技爱好者提供深度、可靠、有价值的内容。

这不是一个营销噱头。这个项目从选题、调研、内容生产、质量检查到数据分析，全部由 AI 完成。人类创始人只负责战略决策和最终审核。

**一句话说清楚我们做什么**：

> 用 AI 探索 AI，把复杂的知识变简单。

### 核心数据一览

| 指标 | 数据 | 说明 |
|------|------|------|
| 🤖 AI 员工 | 14 名 | 覆盖内容、调研、运营全链路 |
| ⏰ 自动化任务 | 12 个/天 | 全自动化运营 |
| 📝 深度调研报告 | 21 份 | 累计 180,000+ 字 |

| 📅 持续运营 | 100 天 | 系统化内容生产 |
| 🔧 开源仓库 | 4 个 | 完整工具链开源 |

### 为什么做这件事

**起源**：2026 年初，AI 技术爆发式发展，但优质内容的供给严重不足。

我们发现三个问题：

**问题一：信息过载**

每天都有新技术、新产品、新论文发布。普通人根本看不过来，更别说理解它们的价值。

**问题二：内容浅层**

大量"AI 热点"内容只是标题党，没有深度分析。读完只知道"有个新东西"，但不知道"这东西有什么用"。

**问题三：门槛过高**

技术文档晦涩难懂，普通用户难以理解。想要真正弄懂一个概念，需要花大量时间查资料。

于是我们想：**能不能用 AI 来解决这些问题？**

AI 擅长收集信息、归纳总结、制作内容。如果让 AI 每天研究一个领域，把复杂的知识用简单的语言传递给普通人，会不会更有价值？

这就是「龙虾巡游记」的起点。

### 我们相信什么

**三大信念**：

```
1. 未来内容应由 AI 生成
   
   大模型擅长收集信息、归纳总结、制作内容，效率远超人类。
   我们相信，未来的优质内容将越来越多地由 AI 创作。

2. 未来平台应对 AI 友好
   
   与其抵制 AI，不如接纳和兼容 AI，探索人机协作的新模式。
   我们相信，AI 友好的平台将在未来获得竞争优势。

3. 赛道选择应发挥 AI 优势
   
   选择 AI 擅长的领域（内容生产、数据分析），而不是让 AI 去做人类更擅长的事。
   我们相信，发挥优势比弥补短板更重要。
```

**我们的定位**：

> 小龙虾巡游发现新的世界，发现很多很好很美妙的东西，然后把新的东西以及领域内的新进展都传播告诉现实世界中的人们。

**核心使命**：发现 · 传播 · 陪伴

## 🚀 Our Work · 核心项目

### 项目一：百日探索计划

**100 天 AI 世界探索之旅**

这是一场系统化的探索——每天研究一个 AI 领域或项目，产出深度内容。

**为什么是 100 天？**

这不是一个随便定的数字。我们调研了内容生产规律后发现：

| 阶段 | 时间 | 目标 |
|------|------|------|
| 第一阶段 | 前 30 天 | 建立内容生产体系，跑通工作流 |
| 第二阶段 | 中 40 天 | 稳定输出，建立用户认知 |
| 第三阶段 | 后 30 天 | 形成品牌影响力，探索商业化 |

100 天足够完成从 0 到 1 的完整周期。

**四大内容板块**：

| 板块 | 内容方向 | 更新频率 | 目标受众 |
|------|----------|----------|----------|
| 🤖 AI 实战 | 工具使用、教程、最佳实践 | 每周 2-3 篇 | 想用 AI 提升效率的人 |
| 🔬 前沿观察 | 技术趋势、行业动态、产品分析 | 每周 2-3 篇 | 关注行业发展的从业者 |
| 📊 数据洞察 | 数据分析、案例研究 | 每周 1-2 篇 | 需要决策依据的管理者 |
| 🛠️ 工具推荐 | 开源项目、效率工具、AI 应用 | 每周 1-2 篇 | 寻找工具的开发者 |

**内容生产流程**：

从选题到发布，全流程自动化：

```
热点发现 → 选题决策 → 深度调研 → 内容生成 → 质量检查 → 多平台发布 → 数据分析
    ↑                                                            ↓
    └────────────────── 反馈优化 ←───────────────────────────────────┘
```

每个环节都有 AI 员工负责，人类只在关键节点进行审核。

**质量保障机制**：

每篇内容都要经过 **5 次检查循环**：

| 检查项 | 标准 | 方法 | 通过条件 |
|--------|------|------|----------|
| 1️⃣ 完整性 | 标题+正文+标签+配图齐全 | 自动脚本 | 100% 齐全 |
| 2️⃣ 去 AI 化 | 自然、流畅、有人味 | AI 评分 | ≥ 4.0/5.0 |
| 3️⃣ 合规性 | 无敏感词、无违规风险 | 规则引擎 | 0 违规 |
| 4️⃣ 原创性 | 100% 原创 | 查重工具 | 查重 < 5% |
| 5️⃣ 可读性 | 通俗易懂、结构清晰 | AI 评分 | ≥ 4.0/5.0 |

只有 5 项全部达标，内容才能发布。

### 项目二：一人公司调研

**深度研究 100 家一人公司**

这是一个长期研究项目——目标是研究全球成功的一人公司案例，提炼可复制的成功模式。

**为什么要研究一人公司？**

2024-2026 年，AI 技术大幅降低了创业门槛。一个人 + AI 工具，就能完成以前需要团队才能做的事。

我们相信，**一人公司**将成为未来的重要趋势。但关于一人公司的深度研究非常少，大多数内容只是简单的案例罗列，缺乏系统化的分析框架。

所以我们决定做这件事——研究 100 家成功的一人公司，找出它们成功的共同规律。

**研究方法论**：

我们建立了一套标准化的研究流程：

```
公司筛选 → 数据收集 → 深度分析 → 模式提炼 → 战略建议
```

**筛选标准**：
- 营收 > $100K/年
- 团队 ≤ 3 人
- 可持续运营 > 2 年

**分析维度**：
- 商业模式：如何赚钱
- 增长策略：如何获客
- 技术栈：用什么工具
- 成功要素：关键决策是什么

**研究价值**：

这些研究不仅是为了输出内容，更重要的是沉淀出一套**可复制的成功模式**：

| 模式 | 核心能力 | 增长路径 | 适用人群 |
|------|----------|----------|----------|
| 技术驱动型 | 独特技术优势 | 开源引流 → 付费转化 | 技术背景创业者 |
| 内容驱动型 | 持续产出优质内容 | 内容引流 → 社群转化 → 产品变现 | 内容创作者 |
| 平台驱动型 | 发现平台红利 | 平台流量 → 私域沉淀 → 多元变现 | 运营型创业者 |

### 项目三：内容生产引擎

**AI 驱动的内容生产流水线**

这是我们最核心的技术能力——一套从选题到发布的全自动化内容生产系统。

**为什么需要自动化？**

传统内容生产的瓶颈：

| 环节 | 传统方式 | 瓶颈 |
|------|----------|------|
| 选题 | 人工判断 | 效率低、覆盖面窄 |
| 调研 | 人工收集 | 耗时长、难高频 |
| 写作 | 人工创作 | 瓶颈明显 |
| 质控 | 依赖经验 | 难标准化 |
| 发布 | 手动操作 | 效率低 |

我们的解决方案：**让 AI 承担 90% 的工作，人类只做 10% 的关键决策**。

**效率对比**：

| 环节 | 传统方式 | AI 自动化 | 提升倍数 |
|------|----------|-----------|----------|
| 选题 | 2-4 小时/天 | 10 分钟 | **12-24x** |
| 调研 | 4-8 小时/篇 | 30 分钟 | **8-16x** |
| 写作 | 2-4 小时/篇 | 15 分钟 | **8-16x** |
| 配图 | 30 分钟/篇 | 2 分钟 | **15x** |
| 发布 | 30 分钟/篇 | 2 分钟 | **15x** |

**核心能力模块**：

| 模块 | 功能 | 技术实现 |
|------|------|----------|
| 🔍 热点发现 | 实时监控全网热点 | 热点 API + AI 判断 |
| ✍️ 内容生成 | 自动生成深度内容 | Claude Sonnet 4.6 |
| 🎨 智能配图 | 自动匹配或生成配图 | Gemini / 即梦 AI |
| ✅ 质量检查 | 5 次检查循环 | 规则引擎 + AI 评分 |
| 📊 数据分析 | 实时分析内容表现 | Pandas + 可视化 |
| 🔄 自动发布 | 多平台自动发布 | Playwright + API |

### 技术栈选型

**为什么选择这些技术？**

我们的技术栈选择遵循三个原则：

1. **成熟稳定**：优先选择经过大规模验证的技术
2. **AI 优先**：优先选择对 AI 友好的技术
3. **开源优先**：优先选择开源技术，便于社区贡献

| 层级 | 技术选型 | 选择理由 |
|------|----------|----------|
| 🧠 大脑层 | Claude Sonnet 4.6 / GLM-5 | 推理能力强、中文友好、成本合理 |
| 🤖 框架层 | OpenClaw | 国产框架、功能完善、社区活跃 |
| 🌐 自动化 | Playwright | 跨浏览器、API 友好、调试工具完善 |
| 🎨 图片生成 | Gemini / 即梦 AI | 质量高、成本低、中文友好 |
| 📊 数据分析 | Python + Pandas | 生态成熟、文档完善 |
| 📱 发布平台 | 小红书创作者平台 | 目标用户集中 |

## 📊 Data Flywheel · 数据飞轮

### 什么是数据飞轮

**核心技术亮点：自动化数据采集与分析闭环**

这是我们的核心技术创新——一个自我进化的数据系统。

传统的内容生产是线性的：选题 → 写作 → 发布 → 结束。

但我们做的是循环的：

```
选题 → 写作 → 发布 → 数据采集 → 分析优化 → 更好的选题 → ...
```

每发布一篇内容，系统就会采集用户反馈数据，分析哪些内容更受欢迎，然后优化下一轮的选题和写作策略。

### 系统架构

```
┌─────────────────────────────────────────────────────────┐
│                    数据飞轮系统                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  📊 数据采集层          🧠 数据分析层                   │
│  ────────────          ────────────                    │
│  · 阅读量               · 内容表现分析                   │
│  · 点赞量               · 用户画像分析                   │
│  · 收藏量               · 时间维度分析                   │
│  · 转发量               · 趋势预测分析                   │
│  · 评论量                                               │
│  · 互动率               ⚡ 决策优化层                   │
│                         ────────────                    │
│  🌐 外部感知层          · 发布频率调整                   │
│  ────────────          · 内容方向优化                   │
│  · 时事热点             · 内容形式改进                   │
│  · 时间因素             · 发布时间优化                   │
│  · 领域进展                                             │
│  · 行业趋势             ✨ 内容输出                     │
│                         ────────────                    │
│                         · 发布新内容                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 数据飞轮的价值

| 维度 | 传统方式 | 数据飞轮 | 提升 |
|------|----------|----------|------|
| 数据采集 | 人工记录，滞后 1-2 天 | 自动采集，实时更新 | **100x** |
| 数据分析 | 凭感觉，无量化 | AI 驱动，精准分析 | **10x** |
| 决策优化 | 月度复盘 | 每日优化 | **30x** |
| 效果验证 | 需要 1-2 周 | 当天可见反馈 | **10x** |

### 实际案例

通过数据飞轮，我们发现了这些规律：

| 发现 | 数据 | 应用 |
|------|------|------|
| 最佳发布时间 | 周二 20:00 发布的内容，互动率比平均高 **35%** | 调整发布时间 |
| 内容形式偏好 | 带案例的内容，收藏率比纯理论高 **50%** | 增加案例比重 |
| 标题最优长度 | 标题 15-20 字的内容，点击率最高 | 优化标题策略 |

这些洞察被自动应用到内容生产流程中，持续提升内容质量。

**10% 人类的工作**：
- 战略方向决策
- 最终内容审核
- 敏感问题处理
- 品牌调性把控

**选题方法论**：

好选题的三个标准：

| 标准 | 说明 | 如何判断 |
|------|------|----------|

| 有价值 | 能解决实际问题 | 用户反馈分析 |
| 有差异 | 提供独特视角 | 竞品内容对比 |

| 合规性 | 无敏感词、无违规 | 规则引擎 | 0 违规 |
| 原创性 | 内容原创 | 查重工具 | 查重 < 5% |
